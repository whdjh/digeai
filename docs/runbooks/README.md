## 작성 기준

장애 한 번 겪었으면 무조건 작성. 다음 사람(미래의 자신 포함)이 새벽에 깨워질 일을 줄이는 게 목적.

## 형식

```markdown
# <시나리오 이름>

## 증상
어떻게 인지하나 (로그, 알림, 사용자 제보 패턴)

## 즉시 조치 (5분)
당장 피해를 멈추는 행동 — 정확한 명령어/UI 경로

## 원인 파악
어느 로그·대시보드를 봐야 하나, 어떤 가설 순서로 좁히나

## 복구
정상화 절차 (옵션이 여러 개면 트레이드오프 명시)

## 사후
postmortem, 재발 방지(모니터링 추가, 코드 수정), 사용자 안내
```

## 원칙

- **명령어는 복붙 가능하게** — 새벽 3시에 머리 안 돌아갈 때를 대비
- **의사결정 트리** — "X면 A, 아니면 B" 분기를 명확히
- **사후 단계 빠뜨리지 마** — 같은 장애 두 번 겪지 않게
