# 세션 로그

날짜별 (`YYYY-MM-DD.md`) 작업 로그. 한 날에 여러 세션이 있어도 한 파일에 시각 헤더로 구분.

## 작성

`/log-session [메모]` 슬래시 커맨드 사용 권장. 오늘 파일에 새 세션 블록 append.

## 템플릿

```markdown
# YYYY-MM-DD

## HH:MM — <세션 주제 한 줄>

### 한 일
- 구체적으로 (파일명/모듈명 포함)

### 결정
- 굳혀진 선택. 큰 결정은 ADR로 따로 빼고 여기엔 링크.

### 이슈 / 다음 액션
- [ ] 미해결 항목 (체크박스)
- [ ] 다음 세션에서 할 것

### 변경 파일
- `path/to/file.js` — 무엇을 바꿨는지
```

## 원칙

- **추측 표시** — 검증 안 한 가설은 "(추정)" 표기
- **다음 액션 명시** — "다음에 뭐 할지 모르겠다"로 끝나지 않게
- **링크 활용** — ADR/runbook/PR/이슈로 이어지면 링크
